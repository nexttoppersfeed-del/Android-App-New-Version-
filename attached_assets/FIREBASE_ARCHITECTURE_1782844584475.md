## 3. Authentication Flow

**Provider:** Google (`GoogleAuthProvider` via `signInWithPopup`)
**Auth State:** Managed by `onAuthStateChanged` in `AuthContext.tsx`

### Flow on Login:
1. User signs in with Google popup → Firebase Auth creates/retrieves user
2. `onAuthStateChanged` fires → `AuthContext` checks `/admins/{uid}` document
3. Role is resolved:
   - `admins/{uid}.role === "owner"` → role = `"owner"`, `isOwner = true`, `isAdmin = true`
   - `admins/{uid}.role === "admin"` → role = `"admin"`, `isAdmin = true`
   - No `admins` doc → role = `"student"`
4. User profile is upserted to `/users/{uid}` via `setDoc(..., { merge: true })`
5. On tab close / `beforeunload`: `isOnline = false`, `lastSeen = serverTimestamp()`

### User Roles:
```
"owner"   → Full admin access, can manage everything
"admin"   → Admin access (subset of owner)
"student" → Default role, read-only for most content
```

## 5. Collection Relationships

```
subjects (1) ──── (many) lecture_folders
subjects (1) ──── (many) lectures
subjects (1) ──── (many) files
lecture_folders (1) ──── (many) lectures   [via folderId]
lecture_folders (1) ──── (many) files      [via folderId]

users (1) ──── (1) leaderboard             [same UID]
users (1) ──── (1) premiumUsers            [same UID]
users (1) ──── (many) testAttempts         [via userId]
users (1) ──── (many) seenAnnouncements    [subcollection]

tests (1) ──── (many) testAttempts         [via testId]

premiumRequests (many) ──── (1) users      [via uid]
premiumRequests (many) ──── (1) privateChatMeta [via chatId]
premiumRequests (many) ──── (1) privateChats/messages [via chatId + msgId]

contactMessages (1) ──── (1) premiumRequests [requestId with "contact_" prefix]

lectures (1) ──── (1) ratings              [via docKey = lectureId_lecture]
files (1) ──── (1) ratings                 [via docKey = fileId_file]

admins (1) ──── (1) users                  [same UID, role sync]
```

## 7. Firebase Cloud Messaging (FCM) & Notifications

### Service Worker
**File:** `artifacts/aarambh/public/firebase-messaging-sw.js`
**Registered in:** `artifacts/aarambh/src/App.tsx` (line ~176)

#### Background Push Handler (`push` event):
- Receives push payload from FCM
- Displays browser notification with:
  - `title`: from `data.notification.title` or default `"📢 New Announcement — NextToppers"`
  - `body`: from `data.notification.body` or default
  - `icon`: `/nexttoppers-logo.png`
  - `badge`: `/nexttoppers-logo.png`
  - `tag`: `data.data.id` or `"announcement"`
  - `data.url`: deep-link URL for notification click

#### Notification Click Handler (`notificationclick` event):
- Closes the notification
- Navigates existing tab to `data.url` or opens new window

### Foreground Notifications Hook
**File:** `artifacts/aarambh/src/hooks/useNotifications.ts`
- Uses `onSnapshot` on the `announcements` collection
- Triggers browser `Notification` API for new announcements in foreground

## 8. Admin & Role System

### Admin Resolution (on every login):
```
GET /admins/{user.uid}
  → exists + role === "owner" → isOwner=true, isAdmin=true, role="owner"
  → exists + role === "admin" → isOwner=false, isAdmin=true, role="admin"
  → does not exist            → isOwner=false, isAdmin=false, role="student"
```

### What Admins Can Do (from `Admin.tsx`):
- Full CRUD on: `subjects`, `lecture_folders`, `lectures`, `files`, `tests`, `announcements`, `coupons`, `yt_channels`
- Manage `premiumUsers` (grant, revoke, extend, edit expiry)
- Review and act on `premiumRequests` and `contactMessages`
- Moderate `communityMessages` (delete)
- Configure `settings/appConfig`, `siteSettings/maintenance`, `siteSettings/boardExam`
- Manage `siteBanners`, `smartBanners`, `subjectPopups`

## 9. Premium / Subscription System

**Context file:** `artifacts/aarambh/src/contexts/PremiumContext.tsx`

### Plans:
| Plan ID | Duration |
|---|---|
| `"day"` | 1 day |
| `"month"` | 30 days |

### Grant Flow (`grantPremium`):
1. Admin approves a `premiumRequest` (or manually grants from Admin panel)
2. `setDoc(/premiumUsers/{uid}, { isPremium: true, plan, activatedAt, expiryTime, expiresAt })`
3. `onSnapshot` on `premiumUsers/{uid}` on user's device detects change instantly

### Revoke Flow (`revokePremium`):
1. `updateDoc(/premiumUsers/{uid}, { isPremium: false })` — triggers instant loss of access

### Expiry Auto-Check:
- On each `onSnapshot` callback in `PremiumContext`:
  - If `isPremium === true` but `expiryTime < now()` → auto-calls `updateDoc({ isPremium: false })`
- **Dual guard:** checks BOTH the `isPremium` flag AND the expiry date

### Payment Proof Flow:
1. Admin sends QR code via private chat → `premiumRequests` doc created with `qrSent: true`
2. User submits transaction ID → `premiumRequests` updated with `proofSubmitted: true`, `transactionId`
3. Payment proof message posted to `privateChats/{chatId}/messages` with `type: "payment_proof"`
4. Admin reviews → approves/rejects → `premiumRequests.status` updated → `premiumUsers` updated

## 10. XP, Leveling & Achievements System

**Context file:** `artifacts/aarambh/src/contexts/XPContext.tsx`

### XP Rules:
| Action | XP Awarded |
|---|---|
| Daily login | `XP_RULES.dailyLogin` |
| Watch a lecture | `XP_RULES.lectureWatch` |
| Read a PDF | `XP_RULES.pdfRead` |
| Quiz completion | `XP_RULES.quizComplete + correctAnswers × XP_RULES.quizCorrectAnswer` |
| Bonus for high score | Additional XP for 80%+ or 100% scores |
| Achievement unlock | Per-achievement `xpReward` (0–1000 XP) |

### Level System (0–100):
| Range | XP per Level |
|---|---|
| Levels 1–5 | 100 XP each |
| Levels 6–10 | 150 XP each |
| Levels 11–20 | 250 XP each |
| Levels 21–30 | 400 XP each |
| Levels 31–50 | 600 XP each |
| Levels 51–70 | 900 XP each |
| Levels 71–90 | 1,400 XP each |
| Levels 91–100 | 2,000 XP each |

### Achievement Categories:
- **Level achievements** (lvl1, lvl5, lvl10, lvl15, lvl20, lvl30, lvl50, lvl70, lvl100)
- **Quiz achievements** (quiz1, quiz10, quiz50, quiz100, perfect1, perfect5, accuracy80)
- **Streak achievements** (streak7, streak30, streak60)
- **Subject achievements** (maths10, maths25, science10, science25, sst10, sst25, + more subjects)
- **Special achievements**

### XP Write Flow:
1. `updateDoc(/users/{uid}, { xp: increment(amount), updatedAt })`
2. `syncLeaderboard(uid)` — reads `/users/{uid}` and writes to `/leaderboard/{uid}`
3. `updateDoc(/users/{uid}, { level: newLevel })`
4. `checkAchievements(uid, stats, existingAchievements)` — unlocks new badges, awards bonus XP

### Daily Limit:
- Daily login XP, lecture XP, and PDF XP are capped once per day using `localStorage` keys (`nt_xp_{key}`)
